# Mooncake 分布式 DRAM 和分布式 SSD 实现详解

## 问题

> Mooncake是怎么实现分布式DRAM，分布式SSD的，在分布式SSD中使用的是哪个文件系统

---

## 回答

### 一、分布式 DRAM 的实现

#### 1.1 架构原理

Mooncake 通过 **Transfer Engine + Segment 抽象** 实现分布式 DRAM：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        分布式 DRAM 实现架构                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   节点 A (Client)                    节点 B (Client)                         │
│  ┌───────────────────┐              ┌───────────────────┐                  │
│  │  本地 DRAM        │              │  本地 DRAM        │                  │
│  │  ┌─────────────┐  │              │  ┌─────────────┐  │                  │
│  │  │ Segment A   │  │◄─── RDMA ───►│  │ Segment B   │  │                  │
│  │  │ (已注册)    │  │              │  │ (已注册)    │  │                  │
│  │  └─────────────┘  │              └───────────────────┘                  │
│  │        ↓          │                      ↓                              │
│  │  registerLocal    │              MountSegment                           │
│  │     Memory()      │              (向 Master 注册)                        │
│  └───────────────────┘                                                     │
│            │                               │                               │
│            └───────────────┬───────────────┘                               │
│                            ↓                                               │
│               ┌─────────────────────────┐                                  │
│               │     Master Service      │                                  │
│               │  • 全局 Segment 元数据   │                                  │
│               │  • 空间分配 (Allocator)  │                                  │
│               │  • 副本位置决策          │                                  │
│               └─────────────────────────┘                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 1.2 关键步骤

**Step 1: 本地内存注册 (Transfer Engine)**
```cpp
// 每个节点在本地注册一块 DRAM 区域
int registerLocalMemory(void *addr, size_t length,
                        const std::string &location,
                        bool remote_accessible,
                        bool update_metadata);
```

**Step 2: 向 Master 挂载 Segment**
```cpp
// 将注册的内存段上报给 Master Service
tl::expected<void, ErrorCode> MountSegment(uint64_t buffer,
                                           uint64_t size,
                                           const std::string& segment_name);
```

**Step 3: Master 统一管理全局资源**
- Master 维护所有节点的 Segment 元数据
- 使用 `BufferAllocator` 管理每个 Segment 的空间分配
- 通过 `AllocationStrategy` 决定数据放置位置

#### 1.3 数据传输机制

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Transfer Engine 支持的传输协议                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────┐         ┌─────────┐         ┌─────────┐                      │
│   │  RDMA   │         │  TCP    │         │ NVMe-oF │                      │
│   │(首选)   │         │(备选)   │         │(SSD)    │                      │
│   └────┬────┘         └────┬────┘         └────┬────┘                      │
│        │                   │                   │                           │
│        └───────────────────┼───────────────────┘                           │
│                            ↓                                               │
│            ┌────────────────────────────────┐                              │
│            │      BatchTransfer API         │                              │
│            │  (统一的读写接口)               │                              │
│            └────────────────────────────────┘                              │
│                                                                             │
│   支持矩阵:                                                                  │
│   ┌──────────────┬────────┬────────┐                                       │
│   │ Remote↓Local→│  DRAM  │  VRAM  │                                       │
│   ├──────────────┼────────┼────────┤                                       │
│   │ DRAM         │   ✓    │   ✓    │                                       │
│   │ VRAM         │   ✓    │   ✓    │                                       │
│   │ NVMe-of      │   ✓    │   ✓    │                                       │
│   └──────────────┴────────┴────────┘                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**核心特性**：
- **零拷贝**: 使用 RDMA 直接在节点间传输，不经过 CPU
- **多网卡池化**: 自动利用多块 RDMA 网卡的聚合带宽
- **拓扑感知**: 根据 NUMA 和 PCIe 拓扑选择最优路径

### 二、分布式 SSD 的实现

#### 2.1 架构原理

Mooncake 的分布式 SSD 是通过 **分布式文件系统 (DFS)** 实现的，而不是直接访问各节点的本地 SSD：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      分布式 SSD (DFS) 实现架构                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   所有节点挂载相同的 DFS 路径 (--root_fs_dir=/mnt/dfs)                        │
│                                                                             │
│   节点 A                    节点 B                    节点 N                 │
│  ┌─────────────┐           ┌─────────────┐           ┌─────────────┐        │
│  │ Client      │           │ Client      │           │ Client      │        │
│  │ /mnt/dfs ───┼───────────┼─ /mnt/dfs ──┼───────────┼─ /mnt/dfs   │        │
│  └──────┬──────┘           └──────┬──────┘           └──────┬──────┘        │
│         │                        │                         │               │
│         └────────────────────────┼─────────────────────────┘               │
│                                  ↓                                         │
│                   ┌───────────────────────────────┐                        │
│                   │    分布式文件系统 (DFS)         │                        │
│                   │                               │                        │
│                   │  ┌─────┐ ┌─────┐ ┌─────┐    │                        │
│                   │  │SSD 1│ │SSD 2│ │SSD N│    │  ← 物理 SSD 分布在      │
│                   │  │节点1│ │节点2│ │节点N│    │    多个存储节点          │
│                   │  └─────┘ └─────┘ └─────┘    │                        │
│                   │                               │                        │
│                   │  支持的文件系统:                │                        │
│                   │  • 3FS (DeepSeek, 推荐)       │                        │
│                   │  • NFS                        │                        │
│                   │  • 任何 POSIX 兼容的 DFS       │                        │
│                   └───────────────────────────────┘                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.2 支持的文件系统

Mooncake 的分布式 SSD 层支持 **两种文件系统接口**：

| 文件系统 | 接口类型 | 性能 | 说明 |
|---------|---------|------|------|
| **3FS** | USRBIO (原生 API) | **最高** | DeepSeek 开发的高性能分布式文件系统 |
| **NFS/Lustre/其他** | POSIX | 标准 | 任何 POSIX 兼容的分布式文件系统 |

#### 2.3 3FS - 推荐的分布式文件系统

**3FS** 是 DeepSeek 开源的高性能分布式文件系统：

```cpp
// hf3fs.h - 3FS 配置
struct Hf3fsConfig {
    std::string mount_root = "/";      // 挂载点根目录
    size_t iov_size = 32 << 20;        // 共享内存大小 (32MB)
    size_t ior_entries = 16;           // IO Ring 最大请求数
    size_t io_depth = 0;               // IO 批处理深度
    int ior_timeout = 0;               // IO 超时 (毫秒)
};
```

**启用 3FS 插件**：
```bash
# 1. 安装 3FS
# https://github.com/deepseek-ai/3FS/

# 2. 编译 Mooncake 时启用 3FS
cmake -DUSE_3FS=ON ...

# 3. 启动 Master 并指定 3FS 挂载点
./mooncake_master --root_fs_dir=/mnt/3fs
```

#### 2.4 数据访问机制

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        分布式 SSD 数据流                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Put 操作 (写入):                                                           │
│   ┌─────────┐  同步写入   ┌───────────────┐                                 │
│   │ Client  │ ─────────→ │ Memory Pool   │ (分布式 DRAM)                    │
│   │         │            │ (via RDMA)    │                                  │
│   │         │  异步写入   └───────────────┘                                 │
│   │         │ ─────────→ ┌───────────────┐                                 │
│   │         │            │ DFS (3FS/NFS) │ (分布式 SSD)                     │
│   └─────────┘            │ 文件名 = key  │                                  │
│                          └───────────────┘                                  │
│                                                                             │
│   Get 操作 (读取):                                                           │
│   ┌─────────┐                                                               │
│   │ Client  │                                                               │
│   │         │──→ 1. 查询 Memory Pool (DRAM)                                 │
│   │         │      ↓                                                        │
│   │         │    命中? → 返回数据                                            │
│   │         │      ↓                                                        │
│   │         │    未命中                                                      │
│   │         │      ↓                                                        │
│   │         │──→ 2. 从 DFS 加载到本地                                        │
│   │         │      ↓                                                        │
│   │         │    返回数据                                                    │
│   └─────────┘                                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 2.5 配置参数

```bash
# Master 启动参数
./mooncake_master \
    --root_fs_dir=/mnt/3fs \           # DFS 挂载目录 (必须所有节点一致)
    --cluster_id=my_cluster \          # 集群 ID (用于隔离)
    --global_file_segment_size=100GB   # DFS 最大可用空间
```

**关键要求**：
- `root_fs_dir` 必须是所有 Client 节点上都有效的 **相同路径**
- 该路径必须是分布式文件系统的挂载点
- 文件名直接使用 KVCache 的 key 作为索引

---

### 三、总结

| 特性 | 分布式 DRAM | 分布式 SSD |
|------|------------|------------|
| **实现方式** | Transfer Engine + RDMA | 分布式文件系统 (DFS) |
| **数据访问** | 直接 RDMA 读写远程内存 | 文件系统 API 读写 |
| **推荐协议** | RDMA (InfiniBand/RoCE) | 3FS USRBIO |
| **备选方案** | TCP | POSIX (NFS/Lustre) |
| **性能** | 极高 (~100GB/s per NIC) | 高 (取决于 DFS) |
| **持久化** | 否 (易失性) | 是 (持久化) |

**回答您的问题**：
1. **分布式 DRAM** 通过 Transfer Engine 实现，每个节点注册本地 DRAM，通过 RDMA 互联形成全局内存池
2. **分布式 SSD** 通过 DFS 实现，推荐使用 **DeepSeek 的 3FS** 文件系统
3. SSD 是 **多节点共享的分布式文件系统**，不是单节点本地 SSD
